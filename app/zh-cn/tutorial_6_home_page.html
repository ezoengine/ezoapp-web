基本组件实作 - 打造入口页面

其实就算是视觉设计师，在做完 UI 交给工程师后，如何把画面按照元搞呈现，往往偏左了些，往往间距大了点，总要一来一往好几次才能确认，而确认后如果又遇到了客户的 UI 需求变更，整个步骤又得重头来过，不仅浪费成本，又几乎把人力资源给耗竭。
现在我们可以利用 EZoApp 这套云端开发工具，运用它提供的现成组件，拖拉进入画面，藉由简单的程序代码编撰 ( 有的画面甚至不需要 )，就可以轻轻松松排列出合适的画面，甚至可以将外部编辑的网页语法贴进编辑器，实时预览产生的画面格式，省下了不少前端与后端衔接的开发时间。 EZoApp 更提供了云端储存的功能，当开发者做好画面后可实时提供给业主或客户进行校稿，同时在云端也提供了完善的版本机制和安全性，因此对于开发者来说更是不可或缺的一套工具。
看过了五个章节的教学介绍，相信大家对于 EZoApp 的威能都有了一定程度的了解吧！接下来的这个章节，我们将用之前所学到的技巧，快速的来打造一个入口画面。

一、直接利用组件来排版

如前几节所述，我们只要利用拖拉的方式，就可以把这些组件一个个的加入到画面当中，程序代码也会自动添加。不过只学会拖拉组件是没有办法在几分钟内完成排版的，这时候我们就要妥善的利用 EZoApp 实用的属性面板工具，来快速的编辑我们所拖曳出来的组件。( 如果找不到属性面版，可以把程序代码区块的大小变小，或先隐藏起来，就可以看到属性面版了 )


二、利用属性面板修改属性

这里我们使用属性面板来编辑 grid 这个组件的属性，我们就可以设定 grid 的字段是一栏还是两栏，若是需要有两列以上，就直接再新增一个 grid 就可以啰！


三、塞入文字

我们同样利用拖拉的方式，就可以轻松的把文字加入到 grid 里头，点选文字，就可以进行编辑，不过这边有个要注意的地方，拖曳进来的文字默认为 h1 的字体大小且靠右对齐，因此我们可以将 h1 标签加入 style=”text-align:center’”，就能够让文字置中啰！ 

<h1 style="text-align:center;">文字</h1>

四、加入图片

关于图片的加载，我们这里使用了3rd Party 里头的 image 组件，不过因为该组件的默认宽度为 100%，高度为 250px，所以拖拉进入画面后会有变形的现象，不过我们只要手动进行修改宽高，并且填入所需的图片网址，就可以轻松的产生合适大小的图片。


五、修改 header 与 footer 的主题样式

header 与 footer 组件，是使用 jQuery Mobile 的基本组件，因此我们只要在 header 和 footer 的标签内，分别加入 data-theme="b" 的一小段程序代码，就可以把样式进行更换。 

<div data-role="header" data-position="fixed" data-theme="b">
    <h3>我是 Header </h3>
</div>

六、万事俱备，只欠东风(背景)

其实到目前为止我们都只是利用拖拉组件，然后修改一下组件的程序代码，就可以把内容排列的差不多了( 不过其实编辑程序代码，还是需要对 CSS 有一些基本的认识，例如 margin 怎么用？height 和 width 的设定...等 )，接下来我们只要挑一张合适的背景图，放在最外层 div 里头当作背景图就可以啰！ 

<div id="home" data-role="page" style="background:url(图片网址); background-size:100% 100%;">
