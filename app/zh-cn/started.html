EZoApp 基本操作介绍

普遍来说，要开发手机的 App，就得使用相关平台的开发程序来进行，若需进行跨平台的开发，更要耗费大量的时间成本，在后续的维护上也是一大挑战。近年来随着浏览器等级与效能的大幅跃进，HTML5 + JavaScript + CSS3 的前端技术窜起，许多前端框架与跨平台展现的语法，有如雨后春笋般的萌生，jQuery Mobile 便是一套建立于 jQuery 与 jQuery UI 的强健基础上，提供行动装置跨平台的用户接口的系统，也是近几年来最夯的技术之一。
在众多编辑器当中，近年来迅速受到开发者喜爱与关注的 EZoApp ，便是由 jQuery Mobile 为基础而诞生的开发工具，EZoApp 不仅一应俱全的保留了 jQuery Mobile 所具备的优势和功能，更结合了 EZo 研发团队多年的开发实战和经验，让使用者在浏览器上工作，撰写简单的语法，打造出无限可能的创意能量和跨平台产品，除此之外，EZoApp 采用云端的运算与储存方式，让开发者更能轻松共享自己的程序代码，设计师也可毫不费力的编辑画面或进行操作体验规划，彼此相得益彰，创造更省时省力的工作方式。
以下将针对 EZoApp 进行基本的操作介绍，让大家体验 EZoApp 前所未有的魅力与感动！

这是 EZoApp 的操作画面，整体的颜色采用程序开发常见的色彩配置 ( 深色底 + 高亮度程序代码 + 高彩度程序代码提示 )，对于工程师而言，这应该是这非常简洁的模式，而对于设计师来说，这就是个所得的开发画面，因为就如同在网页设计工具一样，你写了什么程序代码，拉进去什么图，都会实时的展现在编辑的区域内，相当的方便且实用。
因为画面的编排还满容易上手的，其实不太需要花太多的时间，就可以熟悉如何操作，以下就介绍各个操作区块的功能：

1. 主选单列

在最上方具有五个按钮，就是 EZoApp 的主选单，从左而右分别为：「Device 的尺寸及方向」、「预览模式 / 设计模式」、「改变主题样式」、「在线 Gallery 说明」以及「发布」
鼠标移到第一个「Device 的尺寸及方向」按钮上方，会弹出下拉式选单，除了第一个选项可以把 Device 的方向做水平和垂直的旋转外，其他的都是产生相对应的 Device 尺寸，点选后 Device 的编辑画面就会实时的呈现出来。
点选第二个按钮「预览」，就可以由原本的设计模式，进入发布到手机上的预览模式 ( 按钮的 icon 也会不同 )，同时使用者也可以直接在预览模式检查程序的串接是否正确。
若用户需要不同的组件主题，可以点选这个按钮来进行主题样式的切换，目前主要有两种样式供用户选择，一种是默认的样式 ( Default Style )，一种为扁平化的设计 ( flat style )，当然，用户也可以依据自己的喜好，修改相关的 CSS 就能够调整。
点选 Gallery 的按钮，可以发现 EZoApp 非常贴心的提供了丰富详细的教学说明，除了有二十几种范例之外，更有教学影片提供使用者学习参考。
发布的这个按钮应该是所有开发者最爱的按钮，因为点选这个按钮的时间点，通常都是开发到一定程度或或是项目完成的时间，这个按钮点选后，不仅会带出完整的程序代码让使用者使用，更可以选择是否输出为 apk 的格式，帮使用者做了全方位的服务。

2. 画面编辑区域

编辑区域就如同画布一样，使用者撰写了程序代码之后，在区域上会实时的呈现，使用者也可以直接使用拖拉的方式，将组件拖拉进入编辑区域当中 ( 此时程序代码也会自动产生组件的程序代码 )，目光除了停留在撰写程序代码外，最常将目光焦点摆放的区域就是这个区域了。 虽然说编辑区域司空见惯，但真正有特色的地方，在于当我们对于编辑区域进行了尺寸大小的变换，或是水平垂直方向的变换，内容的组件也都会进行调整，不会因为变换方向、大小或移动，导致程序代码必须重新写，我们甚至可以藉由改变大小或方向，来测试自己的程序或画面是否可以适应各个平台喔！

3. 属性面板

属性面板将编辑区域中各个元素的属性值，忠实且赤裸裸的表现出来，让使用者在不需要撰写程序代码的情况下，惬意的控制组件与画面，而丰富的属性控制，还可以针对页面之间的转换去做选择，直接从繁复的转场效果中挑选合适的效果，甚至也可以选择 icon 样式，对于程序撰写来说，可以大幅缩短开发的时程，同时藉由属性面板与程序代码之间的连动，使用者也能快速学习上手。

4. 组件工具栏

要谈到 EZoApp 的最大亮点，这些组件工具就当之无愧了！身为开发工具的主角，势必拥有相当过人的能力，除了承袭自 jQuery Mobile 的基本组件外，也包含了 EZo 开发团队自行研发的套件 ( 放在 3rd Party 的页签内 )，后面会将这些组件的列表列表，方便大家使用的时候查找。

5. 程序代码编辑区域

EZoApp 的程序编辑器，是程序设计师所熟悉的简洁画面与配色。不过虽然简洁，该有的仍然是应有尽有，例如针对不同程序代码 tag 的颜色提示、不同区块程序代码的收合展现、程序代码字体大小的调配...等，都满足了基本的开发需求，同时，三组分别为 HTML、JS、CSS 的页签也让使用者方便切换，而三种程序代码的颜色也不同喔！高度展现了开发团队的细心与用心良苦。
EZoApp 也因应 jQuery Mobile 版本的变更而做了设计，使用者只要利用下拉选单，就能够切换不同的 jQuery Mobile 版本，如此在编辑上，也较不会因为版本的不同，导致产生的画面有所出入。 还有一点令人感到惊奇，使用者可以自行由 jQuery Mobile 的网站，抓取相关套件的程序代码来贴入编辑，如果是 jQuery Mobile 的官方组件，程序代码贴进来之后，就会变成在编辑区域可以拖拉控制的组件，是不是非常神奇呢！

6. 页面区域

最后介绍剩下的一个区域：Pages，因为 jQuery Mobile 顾名思义是为了开发手机上可以运行的网页或程序接口，而 EZoApp 也是秉持这个精神与架构来设计，因此 Page 就变得格外重要，「页面区域」的编辑面板，提供给使用者一个方便编辑与检索所有页面的区域，用户可以在这个区域浏览成果 ( 就如同 PowerPoint 左侧的页面一样 )，也可以新增、编辑和移动页面，也因为有了页面的工具，编辑起来就会更为顺手和迅速了。
