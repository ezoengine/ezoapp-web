结合 JavaScript 应用 - RSS 服务实作

RSS 的英文全名是 Really Simple Syndication，顾名思义是个非常简单就能整合一切信息的服务，也是个有点像电子报或新闻组的东西，赋予我们非常强大的编辑设计能力，并让我们能简单的获取丰富的数据。
比较常见的范例就像是藉由订阅报章杂志的 RSS，就能够获得最新的文章标题与预览内容，订阅气象局的 RSS 就能掌握最新的气象信息，订阅名人的部落格 RSS，就能无时无刻了解名人的最新文章动态...等，诸如此类，都是 RSS 所带给我们的便利。
过去我们在 RSS 的应用范围，往往都是藉由 RSS 的阅读器或是 email 的通知方式进行，更不用去谈论该如何将这些非常有用的实时信息，应用到行动装置的平台上，而这篇范例，将会介绍如何使用 EZoApp 的 RSS 组件，最快速的产出一系列的文章列表，快速的读取各式报章杂志的 RSS，让我们轻松做出在行动装置平台上的 RSS 阅读程序。

一、建立基本样版

在建立文章清单之前，我们首先要建立一个基本的样板，主要希望点选上方 header 的按钮，下方文章列表可以加载正确的 RSS 内容。
与之前的章节所介绍的作法如出一辙，利用拖拉的方式将 header 、 footer 、 navbar 等组件放入画面当中。


接着我们在 navbar 的程序代码内，进行以下的修改，加入相对应的 RSS 网址，就完成了基本的样板。( RSS 组件默认为 CNN 的 RSS 网址，请自行更换相对应的网址 ) 

<div data-role="navbar" class="menu">

    </li>

二、使用 RSS 组件，建立文章列表

接着我们将 EZoApp 的 RSS 组件拖拉进入画面当中，可以看到自动产生了一个具有标题的列表画面，而列表上许多大括号的代码，是当我们按下 Preview 的按钮之后，会将对应的 RSS 内容放入大括号内。


针对 RSS 组件的属性，除了可以藉由旁边的属性面板，进行最基本的修改，也可以直接修改程序代码编辑器中的 HTML 面板内的程序。 

<div id="gk-428PVoA" is="rss-news" service="http://www.appledaily.com.tw/rss/newcreate/kind/rnews/type/111" rownum="3">
<!--service 请换成第一页的 RSS 网址 -->
<!--rownum 代表产生的文章清单数量 -->
  <ul data-role="listview" data-autodividers="true" data-inset="true" is="json-listview">
    <li divider="{{publishedDate}}" is="listview-li">
      <span>{{title}}</span>
      <br>
      <span style="color:#666;font-size:0.8em;white-space: normal;">{{contentSnippet}}</span>
      <br>
      <span style="color:#666;font-size:0.8em;white-space: normal;">
        <a href="{{link}}" target="_blank">link</a>
      </span>
    </li>
  </ul>
</div>

三、修改 JavaScript 程序代码

画面和 HTML 都完成之后，我们只要对 JavaScript 的程序代码进行修改，就可以轻松的制作出一个新闻的 RSS 阅读器啰！ 


$(document).on("gkComponentsReady", function () {
  var $ele = $("#gk-428PVoA"),
    FEED_URL = $ele.attr("service"),
    $listview = $("#gk-428PVoA").find('[data-role="listview"]');
  rowNum = $ele.attr('rowNum');

  $('.menu li').on('click', function () {
    FEED_URL = $(this).attr('apple');     //点选按钮加载网址
    fn_rss(FEED_URL);
  });

  fn_rss(FEED_URL);     //载入 HTML 上所写第一页的网址

  function fn_rss(FEED_URL) {
    if (FEED_URL) {
      $.ajax({     //利用 JQuery 的 ajax api
        beforeSend: function () {
          $listview.css('visibility', 'hidden');
        },
        url: 'http://ajax.googleapis.com/ajax/services/feed/load?v=1.0&num=' + rowNum + '&callback=?&q=' + encodeURIComponent(FEED_URL),     //利用 google 的 RSS 服务
        dataType: 'json',
        success: function (data) {
          if (data.responseData.feed && data.responseData.feed.entries) {
            var models = data.responseData.feed.entries;
            $listview.gk('model', models);     //利用 EZoApp 的 gk 组件 
            $listview.css('visibility', 'visible');
          }
        }
      });
    }
  }
});

四、属性设定解释

因为 EZoApp 的 RSS 组件应用了 Google 所提供的 RSS 服务，因此在相对应的变量名称上就必须按照 Google 解析出来的名称规则进行 ( 也就是两个大括号内的变量名称 )，以下将列出几个 Google RSS 服务所提供的属性名称：

  1. author：作者信息
  2. content：详细内容，包含标题、内容概要与连结
  3. contentSnippet：内容概要
  4. link：所对应的网址超链接
  5. publishedDate：发布时间
  6. title：文章标题

由于我们可以读取到 RSS 所提供的信息内容，我们也可以自己将内容解析出我们要的区块或信息，就能够进行更弹性的应用与变化，只要善用这些属性，就能够做出专属于自己的 RSS 阅读器啰！
