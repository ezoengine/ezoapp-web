EZoApp 基本操作介紹

普遍來說，要開發手機的 App，就得使用相關平台的開發程式來進行，若需進行跨平台的開發，更要耗費大量的時間成本，在後續的維護上也是一大挑戰。近年來隨著瀏覽器等級與效能的大幅躍進，HTML5 + JavaScript + CSS3 的前端技術竄起，許多前端框架與跨平台展現的語法，有如雨後春筍般的萌生，jQuery Mobile 便是一套建立於 jQuery 與 jQuery UI 的強健基礎上，提供行動裝置跨平台的使用者介面的系統，也是近幾年來最夯的技術之一。
在眾多編輯器當中，近年來迅速受到開發者喜愛與關注的 EZoApp ，便是由 jQuery Mobile 為基礎而誕生的開發工具，EZoApp 不僅一應俱全的保留了 jQuery Mobile 所具備的優勢和功能，更結合了 EZo 研發團隊多年的開發實戰和經驗，讓使用者在瀏覽器上工作，撰寫簡單的語法，打造出無限可能的創意能量和跨平台產品，除此之外，EZoApp 採用雲端的運算與儲存方式，讓開發者更能輕鬆共享自己的程式代碼，設計師也可毫不費力的編輯畫面或進行操作體驗規劃，彼此相得益彰，創造更省時省力的工作方式。
以下將針對 EZoApp 進行基本的操作介紹，讓大家體驗 EZoApp 前所未有的魅力與感動！

這是 EZoApp 的操作畫面，整體的顏色採用程式開發常見的色彩配置 ( 深色底 + 高亮度程式碼 + 高彩度程式碼提示 )，對於工程師而言，這應該是這非常簡潔的模式，而對於設計師來說，這就是個所得的開發畫面，因為就如同在網頁設計工具一樣，你寫了什麼程式碼，拉進去什麼圖，都會即時的展現在編輯的區域內，相當的方便且實用。
因為畫面的編排還滿容易上手的，其實不太需要花太多的時間，就可以熟悉如何操作，以下就介紹各個操作區塊的功能：

1. 主選單列

在最上方具有五個按鈕，就是 EZoApp 的主選單，從左而右分別為：「Device 的尺寸及方向」、「預覽模式 / 設計模式」、「改變主題樣式」、「線上 Gallery 說明」以及「發佈」
滑鼠移到第一個「Device 的尺寸及方向」按鈕上方，會彈出下拉式選單，除了第一個選項可以把 Device 的方向做水平和垂直的旋轉外，其他的都是產生相對應的 Device 尺寸，點選後 Device 的編輯畫面就會即時的呈現出來。
點選第二個按鈕「預覽」，就可以由原本的設計模式，進入發佈到手機上的預覽模式 ( 按鈕的 icon 也會不同 )，同時使用者也可以直接在預覽模式檢查程式的串接是否正確。
若使用者需要不同的元件主題，可以點選這個按鈕來進行主題樣式的切換，目前主要有兩種樣式供使用者選擇，一種是預設的樣式 ( Default Style )，一種為扁平化的設計 ( flat style )，當然，使用者也可以依據自己的喜好，修改相關的 CSS 就能夠調整。
點選 Gallery 的按鈕，可以發現 EZoApp 非常貼心的提供了豐富詳細的教學說明，除了有二十幾種範例之外，更有教學影片提供使用者學習參考。
發佈的這個按鈕應該是所有開發者最愛的按鈕，因為點選這個按鈕的時間點，通常都是開發到一定程度或或是專案完成的時間，這個按鈕點選後，不僅會帶出完整的程式碼讓使用者使用，更可以選擇是否輸出為 apk 的格式，幫使用者做了全方位的服務。

2. 畫面編輯區域

編輯區域就如同畫布一樣，使用者撰寫了程式碼之後，在區域上會即時的呈現，使用者也可以直接使用拖拉的方式，將元件拖拉進入編輯區域當中 ( 此時程式碼也會自動產生元件的程式碼 )，目光除了停留在撰寫程式碼外，最常將目光焦點擺放的區域就是這個區域了。 雖然說編輯區域司空見慣，但真正有特色的地方，在於當我們對於編輯區域進行了尺寸大小的變換，或是水平垂直方向的變換，內容的元件也都會進行調整，不會因為變換方向、大小或移動，導致程式碼必須重新寫，我們甚至可以藉由改變大小或方向，來測試自己的程式或畫面是否可以適應各個平台喔！

3. 屬性面板

屬性面板將編輯區域中各個元素的屬性值，忠實且赤裸裸的表現出來，讓使用者在不需要撰寫程式碼的情況下，愜意的控制元件與畫面，而豐富的屬性控制，還可以針對頁面之間的轉換去做選擇，直接從繁複的轉場效果中挑選合適的效果，甚至也可以選擇 icon 樣式，對於程式撰寫來說，可以大幅縮短開發的時程，同時藉由屬性面板與程式碼之間的連動，使用者也能快速學習上手。

4. 元件工具列

要談到 EZoApp 的最大亮點，這些元件工具就當之無愧了！身為開發工具的主角，勢必擁有相當過人的能力，除了承襲自 jQuery Mobile 的基本元件外，也包含了 EZo 開發團隊自行研發的套件 ( 放在 3rd Party 的頁籤內 )，後面會將這些元件的清單列表，方便大家使用的時候查找。

5. 程式碼編輯區域

EZoApp 的程式編輯器，是程式設計師所熟悉的簡潔畫面與配色。不過雖然簡潔，該有的仍然是應有盡有，例如針對不同程式碼 tag 的顏色提示、不同區塊程式碼的收合展現、程式碼字體大小的調配...等，都滿足了基本的開發需求，同時，三組分別為 HTML、JS、CSS 的頁籤也讓使用者方便切換，而三種程式碼的顏色也不同喔！高度展現了開發團隊的細心與用心良苦。
EZoApp 也因應 jQuery Mobile 版本的變更而做了設計，使用者只要利用下拉選單，就能夠切換不同的 jQuery Mobile 版本，如此在編輯上，也較不會因為版本的不同，導致產生的畫面有所出入。 還有一點令人感到驚奇，使用者可以自行由 jQuery Mobile 的網站，抓取相關套件的程式碼來貼入編輯，如果是 jQuery Mobile 的官方元件，程式碼貼進來之後，就會變成在編輯區域可以拖拉控制的元件，是不是非常神奇呢！

6. 頁面區域

最後介紹剩下的一個區域：Pages，因為 jQuery Mobile 顧名思義是為了開發手機上可以運行的網頁或程式介面，而 EZoApp 也是秉持這個精神與架構來設計，因此 Page 就變得格外重要，「頁面區域」的編輯面板，提供給使用者一個方便編輯與檢索所有頁面的區域，使用者可以在這個區域瀏覽成果 ( 就如同 PowerPoint 左側的頁面一樣 )，也可以新增、編輯和移動頁面，也因為有了頁面的工具，編輯起來就會更為順手和迅速了。
