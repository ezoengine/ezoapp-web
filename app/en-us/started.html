Introduction to the basic operations of EZoApp

In general, to develop apps for cellular phones requires using development programs of related platforms. To develop cross-platform apps requires an even greater amount of time and cost, and poses a huge challenge for ongoing maintenance. In recent years, with the significant leap forward in browser grade and performance, as well as the springing up of the front-end technologies of HTML5 + JavaScript + CSS3, many front-end frameworks and cross-platform display syntaxes were introduced. jQuery Mobile is just such a system that provides a cross-platform user interface for mobile devices which is built upon the solid foundation of jQuery and jQuery UI. It is also one of the most popular technologies in recent years.
Among the many editors, EZoApp, which quickly caught the attention of and loved by developers in recent years, is a development tool born out of the foundation of jQuery Mobile. EZoApp not only includes all the advantages and functions of jQuery Mobile, but also integrates the development experiences of EZo's R&D team. It allows users to work directly in the browser and write simple code to create creative energy and cross-platform products with infinite possibilities. In addition, EZoApp utilizes cloud based computation and storage methods that enable developers to share program code with greater ease, as well as allow designers to easily edit screens or conduct operational experience planning. It thus allows developers and designers to work closely together to facilitate each other's work and create a more efficient and effective work method.
We will introduce the basic operations of EZoApp below. Let's experience EZoApp's unprecedented charm and sensation!

This is the operation screen of EZoApp. The global color scheme adopts the one commonly used for program development (dark background + high brightness code + high saturation code hints). For engineers, this should be a very simple model; and for designers, this is the resulting development screen. Just like in web page design tools, the code you wrote and/or the images you dragged in will be instantly displayed in the editing area. It is quite handy and practical.
Because the layout of the screen is easy to understand, it actually does not take long for you to become familiar with the operation. The features of each operation block are described below:

1. Main menu bar

There are 5 buttons at the very top that make up EZoApp’s main menu. From left to right, they are "Resize", "Preview", "Theme", "Gallery", and "Export".
Move cursor onto the top of the first button, "Resize", and a pull-down menu will appear. The first menu item can rotate the device horizontally or vertically. The rest of the menu items are used to produce corresponding device dimensions; click the menu items will display the device’s editing screen instantly.
Clicking on the second button, "Preview", will switch from the original design mode to the preview mode published to the cell phone (icon of the button will change as well). While in the preview mode, users can also directly check the program connection to see if it is correct.
If users need a different component theme, they can click on the "Theme" button to change the theme style. Currently there are two styles available for users to select. One is the default style (Default Style), and the other is a flat design (Flat Style). Of course, users can modify related CSS to make adjustments based on their own preferences.
Clicking on the "Gallery" button, users can find that EZoApp very thoughtfully provides rich and detailed instructions. In addition to more than 20 examples, there are also instructional videos available as learning references.
The "Export" button should be all developers’ favorite button because the time to click on this button is usually when the development has progressed to a certain stage or when the project has been completed. Clicking on this button will not only bring out the complete program code that is ready to be used, but also allow users to choose whether to export the code as apk format; thus providing a comprehensive range of services to users.

2. Screen editing area

The editing area is just like a drawing canvas. When users have written the code, they will be immediately displayed in this area. Users can also drag components into the editing area directly using the drag-and-drop method (program code for the components will be generated automatically). This is the area on which users most frequently focus their visual attention except for writing program code. Although the editing area is a commonly seen area, the real unique feature is that when we change the size of the editing area, or switch the horizontal/vertical directions, the components contained in the area will also change accordingly; users will not have to re-write the program code because the dimensions and directions have changed, or because of movements. We can even test to see if the program or screen will adapt to various platforms by changing the dimensions or directions.

3. Attribute panel

The attribute panel displays the attributes of each element in the editing area faithfully and plainly so that users can comfortably control components and screens without having to write any program code. The rich attribute control also lets users choose transitions between pages, thus allowing the selection of suitable effects directly from among complicated transition effects. Users can even choose icon styles. It greatly shortens the development time in terms of coding. In addition, users can quickly learn to use it through the linkage between the attribute panel and the program code.

4. Component toolbar

The component tools are certainly the biggest highlight of EZoApp! As the main feature of a development tool, these tools definitely have exceptional capabilities. In addition to the basic components inherited from jQuery Mobile, they also include tool kits developed by the EZo development team (included in the "3rd party" tab). These components will be listed in a table below for easy reference.

5. Program code editing area

EZoApp's program code editor uses the simple screen and color scheme that is familiar to program designers. Although it is simple, it still has all the necessary features to satisfy the basic needs for the development work. For example, color coding for different program code tags, collapse/expansion of code blocks, font sizes, etc. are available. Also, the three tabs of HTML, JS, and CSS allow users to easily switch in between them. The three types of program code also display in different colors! This truly demonstrates the thoughtfulness and attentiveness of the development team.
EZoApp is also designed to work with different versions of jQuery Mobile. Users can switch to a different version of jQuery Mobile simply by using the pull-down menu. Thus, it is less likely to result in producing different screens because of the different versions. There is another surprising feature. Users can go to jQuery Mobile's web site to download the code of related kits to paste in the editing area. If it is jQuery Mobile's official component, after the program code is pasted in, it will become a drag-and-drop component in the editing area. Isn't this magical?

6. Pages panel

Finally, the last area: Pages. Because jQuery Mobile is for developing web pages or a program interface for cellular phones, and EZoApp is also designed in accordance with the same principle and framework, Pages becomes especially important. The editing panel of "Pages panel" provides users with an area to easily edit and retrieve all pages. Users can browse pages created in this area (similar to the panel on the left side in PowerPoint), as well as to create, edit, and move pages. Because of the Pages tool, editing becomes more convenient and quick.
